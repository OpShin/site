      svg = `<svg xmlns="http://www.w3.org/2000/svg" width="18" height="18" viewBox="0 0 24 24" fill="none" stroke="${color}" stroke-width="2" stroke-linecap="round" stroke-linejoin="round" class="feather feather-heart"><path d="M20.84 4.61a5.5 5.5 0 0 0-7.78 0L12 5.67l-1.06-1.06a5.5 5.5 0 0 0-7.78 7.78l1.06 1.06L12 21.23l7.78-7.78 1.06-1.06a5.5 5.5 0 0 0 0-7.78z"></path></svg>`
      break;

    case 'icon-grid':
      svg = `<svg xmlns="http://www.w3.org/2000/svg" width="18" height="18" viewBox="0 0 24 24" fill="none" stroke="${color}" stroke-width="2" stroke-linecap="round" stroke-linejoin="round" class="feather feather-grid"><rect x="3" y="3" width="7" height="7"></rect><rect x="14" y="3" width="7" height="7"></rect><rect x="14" y="14" width="7" height="7"></rect><rect x="3" y="14" width="7" height="7"></rect></svg>`

<section id="faq">
  <h2>FAQ</h2>

  <details open>
    <summary><h3>What is OpShin?</h3></summary>
    <p>
      OpShin is a new <strong>programming language and toolchain for developing
      smart contracts</strong> on the <a href="https://cardano.org/"
      target="_blank">Cardano</a> blockchain. It is focusing on accessability and
      developer experience.
      <br/>
      OpShin is 100% valid Python 3. In this way it can seamlessly integrate
      with off-chain tools built in Python such as <a href="https://pycardano.readthedocs.io/en/latest/">PyCardano</a>.
      It can also leverage the existing testing frameworks and IDEs
      and allows developers to re-use knowledge or focus on attaining
      knowledge about a simple and general purpose language.
    </p>
  </details>

  <details>
    <summary><h3>Why build another smart contract platform?</h3></summary>
    <p>
      Haskell is hard and almost no developer is used to it.
      On the other hand, Python is praised for its simplicity and
      used in almost 25% of all projects on the internet (compare that to roughly 0.65% for Haskell).
      So the ecosystem for Python is huge and full of interesting tools
      that leverage the developer experience.
      The platform based on python can leverage these tools
      to provide a safe and enjoyable developer experience.
    </p>
  </details>

  <details>
    <summary><h3>I thought Cardano smart contracts had to be written in Haskell?</h3></summary>
    <p>
      This is a common misconception. The current Cardano node implementation
      does indeed happen to be written in Haskell. The virtual machine for
      executing smart contracts that comes baked into the node is also
      implemented in Haskell. But that does not mean that it is Haskell which
      is executed by the smart contract virtual machine.
      <br/>
      The virtual machine is a language interpreter which executes a
      smart contract language called <i>'Untyped Plutus Core'</i> (abbrev.
      <code>UPLC</code>) often referred to simply as <i>'Plutus'</i>.
      <br/>
      Yet UPLC isn't something developers are expected to write by hand.
      Instead, it is a compilation target (like WebAssembly for the world wide
      web). Oddly enough, until recently, the only established framework that
      produced UPLC from a high-level syntax was called <i>'Plutus-Tx'</i> and
      happened to be a Haskell framework.
      <br/>
      OpShin changes the game by introducing a new framework that compiles
      straight to UPLC.
    </p>
  </details>

  <details>
    <summary><h3>Can I write off-chain/backend code with OpShin?</h3></summary>
    <p>
      No, but you can use the off-chain library <a href="https://pycardano.readthedocs.io/en/latest/">PyCardano</a>
      with which OpShin is closely integrated!
    </p>
  </details>

  <details>
    <summary><h3>Is Python the right language for smart contracts? It's not even type safe right?</h3></summary>
    <p>
      We chose Python because it is simple and intuitive.
      This is a great basis to improve security of contracts -
      if they are easy to read and understand, a larger group of people can interpret them.

      To combat the problems that come with dynamically typed languages,
      OpShin enforces a strict type system on top of Python, similar to the Haskell type system.
      Hence, you enjoy the simplicy of Python programming and the safety of properly typed code.
    </p>
  </details>
